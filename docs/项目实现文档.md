## 1. 仓库与职责

| 仓库 | 路径（本地） | 职责 |
|------|--------------|------|
| 主站 | `safebase_front_cursor` | React SPA：注册登录、对话、日记、注销 |
| 后端 | `safebase_backend_cursor` | Fastify API、RAG、批处理、Schema、Docker Compose |
| 管理端 | `safebase_admin_cursor` | 只读运营台：用户列表与详情 |

三者 **独立 Git 仓库**，共享同一数据库 `trauma_heal` 与同一套 `/api/*`。

---

## 2. 技术栈

| 层 | 技术 |
|----|------|
| 主站 | React 18、TypeScript、Vite 5、Ant Design 5、Zustand、react-hook-form + zod、react-markdown |
| 管理端 | React 18、Vite 5、Ant Design、Axios、Zustand persist |
| 后端 | Node ≥18、TypeScript、Fastify 5、`pg`、`jose`（JWT）、`bcryptjs` |
| 数据库 | PostgreSQL 16 + pgvector（Docker 镜像 `pgvector/pgvector:pg16`） |
| 对话 LLM | 默认 **DeepSeek 官方**（`deepseek-reasoner` / `deepseek-chat`）；可切 OpenRouter |
| Embedding | **始终 OpenRouter**（默认 `openai/text-embedding-3-large`，2048 维） |

## 5. 鉴权实现

| 客户端 | 凭证 | 校验 |
|--------|------|------|
| 主站用户 | JWT（localStorage `safebase_token`，约 7 天） | `Authorization: Bearer` → `requireUser` |
| 管理端 | 静态密钥 | `X-Admin-Key` === `ADMIN_SECRET` |

- 密码：bcrypt，12 rounds  
- JWT：`jose` HS256，payload 含 `sub`（user UUID）、`username`  
- **无 Postgres RLS**：行级隔离由 API 层强制 `user_id = request.userId`

## 6. 对话链路（核心）

```text
前端 ChatPage
  1) POST /api/messages { role:user, content } → user_message_id
  2) POST /api/chat/stream { messages, user_message_id }
       ├─ 校验消息归属
       ├─ 更新用户消息 embedding（OpenRouter）
       ├─ resolveChatModel() → deep | fast
       ├─ buildChatMessages()：画像 + 近期对话 + 摘要匹配 + 锚点匹配
       ├─ renderChatMessages()：system + user 模板
       ├─ 上游 chat/completions（SSE）→ 代理给前端
       └─ insertAssistantMessage() + embedding
```

### 6.1 Prompt 组装

文件：`src/chat/prompt.ts`

- **System**：疗愈伴侣人设与目标（说透、看清、篇幅随复杂度展开）
- **User 模板**：`[用户画像] [近期对话] [历史摘要] [重要锚点] [当前输入]`
- 已移除 user 侧「intake 任务块」（泛化差）

### 6.2 模型路由

文件：`src/chat/model-router.ts`（`OPENROUTER_CHAT_ROUTING=true` 时）

| 轨 | 触发 | 默认模型（DeepSeek） | max_tokens |
|----|------|----------------------|------------|
| deep | 字数 ≥ `OPENROUTER_CHAT_DEEP_MIN_CHARS`（默认 300）或深度关键词 ≥2 | `deepseek-reasoner` | 16384 |
| fast | 其他 | `deepseek-chat` | 3072 |

深度关键词示例：NPD、创伤、闪回、解离、依恋、价值体系、茶饭不思、夜不能寐、年份 2018–2025 等。

### 6.3 RAG / 记忆

文件：`src/chat/memory.ts`

| 来源 | 机制 |
|------|------|
| 画像 | `profiles.content` |
| 近期对话 | RPC `get_recent_memory_messages`（约 30 条） |
| 日摘要 | RPC `match_summaries_daily`（向量 Top-K） |
| 锚点 | RPC `match_anchors`（向量 Top-K） |

说明：Schema 存在 `match_diaries`，**当前实时聊天未接入日记向量检索**；日记 embedding 供批处理与后续扩展。

## 7. 批处理任务

入口：`npm run tasks -- daily|profiles|anchors`（`scripts/run-tasks.ts`）

| 任务 | 作用 |
|------|------|
| `daily` | 根据昨日消息生成 `summaries`（type=daily） |
| `profiles` | 更新用户 Markdown 画像（核心画像 / 触发清单 / 资源库） |
| `anchors` | 维护重要事件锚点（可更新演化历史、抽取新事件） |

生产用 cron（示例见 `scripts/cron.example`），建议夜间错峰执行。提示词可覆盖：`prompts/*.txt`。

## 8. 数据模型（摘要）

| 表 | 用途 |
|----|------|
| `users` | 账号：username、email（合成）、password_hash |
| `profiles` | 一人一行 Markdown 画像 |
| `messages` | 对话；`role` user/assistant；`embedding vector(2048)` |
| `diaries` | 日记 + embedding |
| `summaries` | 周期摘要（当前实现以 daily 为主） |
| `anchors` | 事件名、初始/当前想法、evolution_history jsonb、embedding |
| `data_access_audit` | 审计表结构已建，**应用层暂无写入路径** |

完整 DDL：`safebase_backend_cursor/sql/migrations/001_initial.sql`。

## 10. 配置项（实现相关）

权威示例：`safebase_backend_cursor/.env.example`

关键项：

- `DATABASE_URL`、`JWT_SECRET`、`ADMIN_SECRET`
- `LLM_CHAT_PROVIDER=deepseek|openrouter`
- `DEEPSEEK_*` / `OPENROUTER_*`（含 deep/fast 模型名、routing、max_tokens）
- Embedding：`OPENROUTER_API_KEY` + `OPENROUTER_EMBEDDING_*`

主站 / 管理端：`VITE_API_BASE_URL` 可空（同源 `/api`）。
