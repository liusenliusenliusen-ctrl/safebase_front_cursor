## 1. 部署架构

```text
用户浏览器
  ├─ http://服务器/              → Nginx → /opt/safebase/front
  │     /api/*                   → proxy → 127.0.0.1:8000
  ├─ http://服务器:8081/         → Nginx → /opt/safebase/admin
  │     /api/*                   → proxy → 127.0.0.1:8000
  └─ SSE /api/chat/stream        → 同上（须 proxy_buffering off）

/opt/safebase/backend
  ├─ PM2: safebase-backend (:8000)
  ├─ Docker: trauma-heal-postgres (宿主机 :5433)
  └─ cron → node dist/scripts/run-tasks.js
```

推荐主站与 API **同源**（Nginx 反代 `/api`），前端构建时不必设置 `VITE_API_BASE_URL`。

## 2. 服务器要求

| 项 | 建议 |
|----|------|
| 规格 | ≥2 vCPU + 4 GiB；建议配置 swap |
| 系统 | Linux；Docker Compose v2；Node 18+；Nginx；PM2 |
| 端口 | 80（主站）、8081（管理端）；**5433/5432 勿对公网开放** |
| 出站 | 可访问 DeepSeek API、OpenRouter API |

服务器目录约定：

```text
/opt/safebase/front      # 主站静态文件
/opt/safebase/admin      # 管理端静态文件
/opt/safebase/backend    # Node 产物、sql、docker-compose、prompts、.env
```

## 3. 本地开发启动（常测）

三仓库并列放在同一父目录下。

```bash
# 1. 数据库
cd safebase_backend_cursor
docker compose up -d
# 确认：docker ps | grep trauma-heal-postgres

# 2. 后端（先配好 .env）
cp .env.example .env   # 首次
# 编辑 .env：JWT_SECRET、DEEPSEEK_API_KEY、OPENROUTER_API_KEY、ADMIN_SECRET 等
npm install
npm run dev            # :8000

# 3. 主站
cd ../safebase_front_cursor
npm install
npm run dev            # :5173

# 4. 管理端（可选）
cd ../safebase_admin_cursor
npm install
npm run dev            # :5174
```

验证：

```bash
curl -s http://127.0.0.1:8000/api/health   # {"ok":true}
# 浏览器 http://127.0.0.1:5173/auth
# 管理端 http://127.0.0.1:5174/login ，密钥 = ADMIN_SECRET
```

后端占用端口时：

```bash
lsof -ti :8000 | xargs kill -9
cd safebase_backend_cursor && npm run dev
```

## 4. 后端环境变量（生产 / 本地）

文件位置：**`/opt/safebase/backend/.env`**（与 `package.json` 同级）。**不要**打进部署包、不要提交 Git。

最小示例（按实际密钥填写）：

```env
DATABASE_URL=postgresql://postgres:强密码@127.0.0.1:5433/trauma_heal
JWT_SECRET=至少32字符的随机串
ADMIN_SECRET=管理后台密钥

LLM_CHAT_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-...
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_CHAT_MODEL_DEEP=deepseek-reasoner
DEEPSEEK_CHAT_MODEL_FAST=deepseek-chat

OPENROUTER_API_KEY=sk-or-v1-...
OPENROUTER_BASE_URL=https://openrouter.ai/api/v1
OPENROUTER_EMBEDDING_MODEL=openai/text-embedding-3-large
OPENROUTER_EMBEDDING_DIMENSIONS=2048
OPENROUTER_CHAT_ROUTING=true
OPENROUTER_CHAT_DEEP_MIN_CHARS=300
OPENROUTER_CHAT_MAX_TOKENS_DEEP=16384
OPENROUTER_CHAT_MAX_TOKENS_FAST=3072

PORT=8000
HOST=0.0.0.0
```

说明：

- **对话**默认走 DeepSeek；**Embedding** 仍依赖 OpenRouter。  
- 完整键名见 `safebase_backend_cursor/.env.example`。  
- 修改 `.env` 后必须：`pm2 restart safebase-backend --update-env`。

## 5. 一键部署（推荐）

在 **本机 Mac**、已配置 SSH 到服务器时：

```bash
cd safebase_front_cursor
bash scripts/deploy-upload.sh
# 或指定机器：
# DEPLOY_SERVER=root@你的IP bash scripts/deploy-upload.sh
```

脚本会：

1. 构建 front / admin / backend  
2. 打包（`COPYFILE_DISABLE=1`，避免 macOS xattr）  
3. `scp` 到服务器 `/tmp/`  
4. 执行远程 `deploy-server.sh`：解压到 `/opt/safebase/*`、保留 `.env`、`docker compose up`、`npm ci --omit=dev`、PM2 重启、Nginx reload  

**首次部署前**须先把 `.env` 放到服务器：

```bash
scp safebase_backend_cursor/.env root@118.196.122.208:/opt/safebase/backend/.env
```

（若目录尚不存在，可先手动 `mkdir -p /opt/safebase/backend`。）

## 6. 数据库

```bash
ssh root@118.196.122.208
cd /opt/safebase/backend
docker compose up -d
docker compose ps          # healthy
```

- 容器名：`trauma-heal-postgres`  
- 默认库：`trauma_heal`  
- 宿主机端口：**5433**（避开本机 5432）  
- 生产务必改 `POSTGRES_PASSWORD`，并同步 `DATABASE_URL`  
- 新 migration：放入 `sql/migrations/`；**已有 volume 不会自动重跑**，需手动 `psql` 执行或重建库  

本地清空全部用户与数据（慎用）：

```bash
docker exec trauma-heal-postgres psql -U postgres -d trauma_heal -c \
  "DELETE FROM public.messages; DELETE FROM public.diaries; DELETE FROM public.summaries; DELETE FROM public.anchors; DELETE FROM public.profiles; DELETE FROM public.data_access_audit; DELETE FROM public.users;"
```

仅清空业务数据、保留用户：backend 仓库 `npm run clear-data`。

## 8. Nginx 要点

| 站点 | root | 反代 |
|------|------|------|
| 主站 `:80` | `/opt/safebase/front` | `/api/` → `http://127.0.0.1:8000` |
| 管理 `:8081` | `/opt/safebase/admin` | 同上 |

务必：

- SPA：`try_files $uri $uri/ /index.html;`  
- 流式：`location /api/` 内 `proxy_buffering off;`（及合适的 `proxy_read_timeout`）  
- **root 是 `/opt/safebase/front`，不是 `front/dist`**（部署脚本已把 dist 内容解到该目录）

改完配置：`nginx -t && systemctl reload nginx`。

## 9. 批处理 Cron

参考 `safebase_backend_cursor/scripts/cron.example`，在服务器 crontab 中配置，例如：

```cron
# 示例（按服务器时区调整）
30 23 * * * cd /opt/safebase/backend && node dist/scripts/run-tasks.js daily >> /var/log/safebase-tasks.log 2>&1
10 0 * * * cd /opt/safebase/backend && node dist/scripts/run-tasks.js profiles >> /var/log/safebase-tasks.log 2>&1
30 0 * * * cd /opt/safebase/backend && node dist/scripts/run-tasks.js anchors >> /var/log/safebase-tasks.log 2>&1
```

手动跑一轮：

```bash
cd /opt/safebase/backend
node dist/scripts/run-tasks.js daily profiles anchors
```

## 11. 常见故障

| 现象 | 排查 |
|------|------|
| `/auth` 打不开 | 前端未起或 Nginx root 指错 |
| `EADDRINUSE :8000` | `lsof -ti :8000 \| xargs kill -9` 后重启 |
| 注册 500 / JWT 相关 | 缺 `.env` 或未 `--update-env` |
| 对话无流式 / 卡住 | 查 `DEEPSEEK_API_KEY` / `OPENROUTER_API_KEY`；`pm2 logs`；Nginx `proxy_buffering` |
| 管理端 401 | `ADMIN_SECRET` 与输入不一致，或后端未加载 `.env` |
| DB 角色不存在 / 连错库 | 误连本机 5432；应使用 Docker **5433** |
| 解压 macOS 警告 | 打包用 `COPYFILE_DISABLE=1 tar`（部署脚本已含） |

---

## 12. 安全注意

- 切勿把 `.env`、真实密钥提交 Git 或打进 tar  
- Postgres 端口仅绑定本机  
- 生产修改默认 `postgres/postgres` 密码  
- 管理端密钥等同高权限；勿写进前端仓库  
- 当前 admin 可读用户消息：仅限受信运维，后续按 `SECURITY_EVOLUTION.md` 收敛
