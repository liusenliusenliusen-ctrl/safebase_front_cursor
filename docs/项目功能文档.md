## 1. 产品功能总览

```text
主站（用户）
├── 注册 / 登录
├── 疗愈对话（流式 + 历史 + 停止）
├── 个人日记
└── 注销账户

管理端（内部）
├── 密钥登录
├── 用户列表与用量统计
└── 用户详情（画像 + 近期消息）

系统（后台）
├── RAG 记忆注入
├── 消息 / 日记向量化
└── 夜间批处理（摘要 / 画像 / 锚点）
```

## 2. 主站功能

### 2.1 注册与登录

| 功能点 | 说明 |
|--------|------|
| 入口 | `/auth`，登录 / 注册 Tab |
| 字段 | 用户名、密码 |
| 校验 | 用户名 3–64 字符；密码 6–128 字符 |
| 结果 | 写入 JWT；跳转聊天页 |
| 品牌文案 | SafeBase / 创伤疗愈伴侣；「在这里，你被看见、被接纳」 |

### 2.2 疗愈对话

| 功能点 | 说明 |
|--------|------|
| 入口 | `/`（需登录） |
| 发送 | 输入框发送；展示用户气泡 + 助手 Markdown 流式输出 |
| 历史 | 默认拉取最近消息；上滑加载更早记录 |
| 停止 | 中止流式；撤销本轮用户消息；内容回到输入框 |
| 伴侣能力 | 理解创伤语境；结合画像/摘要/锚点做连续陪伴（用户无感知底层 RAG） |
| 深度分流 | 长文或含深度信号 → 深度模型；短句 → 快轨（用户侧无开关） |

**伴侣定位（功能语义）**：具备深度洞察的陪伴者——坚定理解与温暖承接；把困扰说透、看清；帮助认识自己并看见成长。

### 2.3 个人日记

| 功能点 | 说明 |
|--------|------|
| 入口 | `/diary` |
| 列表 | 分页；支持关键词搜索（本地对批量结果过滤） |
| 编辑 | 抽屉内编辑标题与正文；支持 Cmd/Ctrl+S 保存 |
| 删除 | 确认后删除 |
| 后台效果 | 创建/更新后异步生成 embedding，供记忆体系使用 |

### 2.4 账户操作

| 功能点 | 说明 |
|--------|------|
| 位置 | 顶栏用户菜单 |
| 退出登录 | 清除本地 JWT |
| 注销账户 | 确认后删除该用户全部服务端数据（不可恢复） |

### 2.5 明确没有的页面

- 独立「设置」页  
- 用户手动编辑「画像 / 摘要 / 锚点」  
- 多会话列表（当前为**单线程连续对话**）

## 3. 管理端功能

| 功能点 | 说明 |
|--------|------|
| 地址（本地） | `http://127.0.0.1:5174` |
| 地址（生产） | `http://<服务器>:8081` |
| 登录 | 输入与后端 `ADMIN_SECRET` 一致的密钥 |
| 用户列表 | 用户名、ID、注册时间、消息数 / 摘要数 / 锚点数 |
| 用户详情 | 画像全文、近期对话内容、统计 |
| 权限 | **只读**；不能代发消息、改日记、改画像 |

> 注意：当前阶段为便于调试，详情页可展示消息正文。隐私收敛计划见 `SECURITY_EVOLUTION.md`。

## 4. 系统功能（对用户透明）

### 4.1 实时记忆（每次对话）

系统自动拼入上下文：

1. 用户画像  
2. 近期对话片段  
3. 相关日摘要（向量召回）  
4. 相关事件锚点（向量召回）  

### 4.2 异步记忆建设

| 任务 | 用户可感知结果 |
|------|----------------|
| 日摘要 | 隔日后回复更可能「记得」昨天主题 |
| 画像更新 | 长期模式、触发点、资源被总结进画像 |
| 锚点维护 | 重要事件与想法演化被结构化保留 |

定时执行依赖服务器 cron（见部署文档），非用户点击触发。

## 5. API 功能对照（用户侧）

| 能力 | 方法 | 路径 |
|------|------|------|
| 注册 | POST | `/api/auth/register` |
| 登录 | POST | `/api/auth/login` |
| 当前用户 | GET | `/api/auth/me` |
| 消息列表 | GET | `/api/messages` |
| 写入消息 | POST | `/api/messages` |
| 删除最后用户消息 | DELETE | `/api/messages/last-user` |
| 流式对话 | POST | `/api/chat/stream` |
| 日记列表 / 批量 | GET | `/api/diaries`、`/api/diaries/batch` |
| 日记增改删 | POST / PATCH / DELETE | `/api/diaries`、`/api/diaries/:id` |
| 注销 | DELETE | `/api/account` |
| 健康检查 | GET | `/api/health` |

管理端：

| 能力 | 方法 | 路径 | 鉴权 |
|------|------|------|------|
| 用户列表 | GET | `/api/admin/users` | `X-Admin-Key` |
| 用户详情 | GET | `/api/admin/users/:userId` | `X-Admin-Key` |

## 6. 功能验收清单（冒烟）

- [ ] 注册新用户并进入聊天页  
- [ ] 发送短消息，收到温暖短回复  
- [ ] 发送长自述，进入深度回复（耗时更长、内容更展开）  
- [ ] 停止生成后，用户消息被撤销、可改写重发  
- [ ] 上滑加载历史消息  
- [ ] 创建 / 编辑 / 删除一篇日记  
- [ ] 管理端密钥登录，能看到该用户计数与详情  
- [ ] 注销账户后无法再登录，数据不可见
